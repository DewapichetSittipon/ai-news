# Aggregate multiple AI labs, feed-first, via per-source adapters

The reader covers several AI labs (Anthropic, OpenAI, Google DeepMind, Google AI, Hugging Face), not just Anthropic. Each Source is pulled by an **adapter** (`scripts/adapters/`) and merged into one Snapshot, so adding a Source is a config entry in `scripts/sources.ts` plus, if its shape is unusual, an adapter. Every News Item carries a `source` and a composite `id` (`${source}__${slug}`) used for routing and per-article filenames, since slugs can collide across labs.

We prefer **official RSS/Atom feeds** over scraping: OpenAI, DeepMind, Google AI and Hugging Face all publish feeds, which are far less fragile than HTML scraping. Anthropic has no feed, so it keeps its bespoke scraper (ADR-0002). Deliberately deferred: labs whose sites are JS-rendered SPAs or bot-blocked (DeepSeek, Z.ai, Qwen, xAI). Scraping those needs a headless browser and breaks often; when wanted they should come through a feed bridge (e.g. RSSHub) rather than bespoke Playwright scrapers. Trade-off accepted: feed content quality varies — some feeds give full article bodies, others only summaries, so a News Item's in-app body can be thinner than the original, which the "read original" link always backstops.
